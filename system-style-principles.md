## 整体设计原则与系统概览

- **目标**：为整套 Style Guide 网站提供统一的视觉与代码风格标准，提升可读性、一致性和可维护性。
- **核心理念**：
  - **一致性优先**：相同语义使用相同结构和样式，让读者集中在内容本身，而不是表现形式。
  - **可读性与可扫描性**：文本、示例代码、标题层级和表格都便于快速扫读和对比。
  - **语义化与可访问性**：HTML 结构遵循语义化；为多媒体和图片提供合适的替代文本。
  - **分离关注点**：结构（HTML）、表现（CSS）、行为（JS）严格分离，减少耦合，方便维护。
  - **标准与有效性**：尽可能使用符合标准的 HTML/CSS，并使用校验工具保证质量。

## 布局与排版规范

- **页面布局**
  - **背景色**：`#ffffff`，保持内容区域干净简洁。
  - **文字颜色**：主体文本 `#333333`，对比度较高，适合长时间阅读。
  - **页面边距**：左右各约 100px 的留白，使内容居中且不贴边。

- **字体与字号**
  - **基础字体**：`sans-serif`，10pt 作为通用正文字号。
  - **标题层级**：
    - `h1`：18pt，居中，对应文档主标题。
    - `h2`：12pt，加粗，用于一级章节标题。
    - `h3–h6`：10pt，加粗或保持与正文接近，用于子章节和小节。
  - **标题缩进**：
    - `h2` 及 `.toc_title`：左侧有一定“负缩进”（向左突出），强化章节层级感。
    - `h3–h6`：较小的负缩进，视觉上与 `h2` 形成层级。

- **表格**
  - 使用 `border-collapse: collapse` 保持边框合并。
  - 单元格 `td, th` 使用 `1px solid #ccc` 的边框和适度内边距（2px 12px）。
  - 目录表格（`.toc`）在左右边框加粗（`border-width: 1px 5px`），突出导航区域。

## 颜色系统

- **基础色**
  - **背景白**：`#ffffff`（`body` 背景）。
  - **正文深灰**：`#333333`（普通文本）。

- **品牌与强调色**
  - **主色蓝**：`#0066cc`（标题 `h1–h6` 与 `.toc_title`），用于核心信息和结构标题。
  - **章节标签蓝**：`#58f`（`.stylepoint_section`），用于大章节小标签。
  - **子章节蓝灰**：`#679`（`.stylepoint_subsection`、`.stylepoint_subsubsection`），用于层级较低的标签与提示文字。

- **代码与状态色**
  - **代码绿色**：`#006600`（`code, samp, var, pre`），表示“推荐/正常示例代码”。
  - **代码背景浅绿**：`#f8fff8`（普通 `pre` 背景），弱化区块同时区分正文。
  - **代码边框浅绿**：`#f0fff0`（普通 `pre` 边框），与背景协调。
  - **错误/反例红**：`#cc0000`（`.badcode` 文本），用于“Not recommended / bad 示例”。
  - **错误背景浅红**：`#fff8f8`（`.badcode` 背景）。
  - **错误边框浅红**：`#fff0f0`（`.badcode` 边框）。

- **控件与辅助色**
  - **链接按钮背景浅蓝**：`#f8f8ff`（`.link_button` 背景）。
  - **链接按钮边框浅蓝**：`#f0f0ff`（`.link_button` 边框）。
  - **分割线**：`hr` 使用浅色细线（1px），增加段落之间的呼吸感。

## 组件与样式规范

### 标题与章节组件

- **文档主标题（`h1`）**
  - 内容：文档名称（如 “Google HTML/CSS Style Guide”）。
  - 样式：居中、18pt、主色蓝 `#06c`。
  - 使用场景：页面最顶部，仅出现一次。

- **章节标题（`h2` 与 `.toc_title`）**
  - 内容：一级章节（如 “Background”、“General”、“HTML”、“CSS”等）。
  - 样式：加粗、12pt、主色蓝，左侧负缩进突出。
  - 行为：通常带有 `id` 和 `.numbered` 用于生成目录和锚点。

- **子章节标题（`h3–h6`）**
  - 内容：更细粒度的规则分类（如 “Indentation”、“Capitalization”、“Comments”等）。
  - 样式：10pt、主色蓝，缩进略小于 `h2`。
  - 行为：用于页面内导航和快速扫描。

- **规则标签（Stylepoint）**
  - `.stylepoint_section`：标记一类规则的大章节，颜色 `#58f`，字体略小于正文且加粗。
  - `.stylepoint_subsection`：子级规则标签，颜色 `#679`，加粗。
  - `.stylepoint_subsubsection`：再下一层级的规则标签，字体更小，颜色 `#679`。
  - 使用场景：配合目录与编号，将风格点分组展示。

### 文本与代码组件

- **正文段落（`p`）**
  - 统一使用 10pt、深灰 `#333`。
  - 用于说明规则的背景、原因与示例前的解释性文字。

- **行内代码（`code, samp, var`）**
  - 颜色 `#060`，与正文明显区分。
  - 使用语境：展示 HTML 标签、属性、CSS 属性名/值、关键字（如 `TODO`、`<doctype html>` 等）。

- **代码块（`pre` + `code`）**
  - 普通示例代码：
    - 文本色：`#060`。
    - 背景：`#f8fff8`。
    - 边框：`1px solid #f0fff0`，左边框加粗（5px），突出代码块。
    - 内边距：`4px 12px`，方便阅读和拷贝。
  - 反例代码（`.badcode`）：
    - 文本色：`#c00`。
    - 背景：`#fff8f8`。
    - 边框：`1px solid #fff0f0`。
  - 用法约定（来自 HTML/CSS 指南）：
    - 同一规则下通常配对展示 **bad** 与 **good** 示例。
    - bad 示例用于“不要这样写”，good 示例是“推荐写法”。

### 表格组件

- **通用表格（`table` + `td`/`th`）**
  - 使用细边框和适度内边距提高可读性。
  - 多用于展示属性对照表、规则对比或结构示意。

- **目录表格（`.toc`）**
  - 左右边框加粗，以区分于普通表格。
  - 配合 `.toc_title`、`.toc_category`、`.toc_stylepoint` 展现多层级目录。

### 导航与元信息组件

- **目录项（`.toc_category`, `.toc_stylepoint`）**
  - 字号统一为 10pt，行距通过上下 padding 控制。
  - 功能：承载章节结构，跳转到相应规则位置。

- **修订信息（`.revision`）**
  - 文本右对齐。
  - 用于展示文档版本、更新时间或维护者信息。

- **页脚地址（`address`）**
  - 右对齐。
  - 一般用于联系信息或版权声明。

## 交互与行为规范

### 展示/收起按钮（`.showhide_button`）

- **视觉样式**
  - 浮动在左侧（`float: left`），通常紧邻相关内容或标题。
  - 边框为浅色立体感效果：`border-color: #ddd #aaa #aaa #ddd`。
  - 较小的圆角（`border-radius: 3px`）提升现代感。
  - 内边距与外边距确保点击区域舒适（左右内边距 3px，底部外边距 8px）。

- **交互行为**
  - `cursor: pointer` 明确为可点击元素。
  - 典型用法：展开/收起某一段说明、规则细节或示例代码块。
  - 建议交互：
    - 点击后切换按钮文案（如 “Show” / “Hide”）。
    - 切换关联内容区域的显示/隐藏状态。

### 链接按钮（`.link_button`）

- **视觉样式**
  - 浮动在左侧，与 `.showhide_button` 相邻。
  - 默认 `display: none;`，通过 JS 按条件显示。
  - 背景浅蓝 `#f8f8ff`，边框 `#f0f0ff`，字体为 75% 大小，作为次级控件。

- **交互行为**
  - 用于提供“复制链接”、“跳转到此规则”等辅助操作。
  - 显示逻辑通常由 JS 控制，例如：
    - 鼠标悬停在某个标题区域时显示。
    - 页面加载后根据是否支持相关功能决定是否显示。

### 链接与锚点

- 标题、规则和章节通常带有 `id` 并配合 JS 生成可链接的锚点。
- 链接按钮或目录点击后滚动到对应位置。
- 建议行为：
  - 平滑滚动或快速跳转，保持用户对当前位置的感知。
  - 为键盘和屏幕阅读器用户提供可访问的焦点管理。

## HTML/CSS 编码风格原则（来自 HTML/CSS 指南）

### 通用规则

- **协议**：在可能的情况下使用 HTTPS，为脚本、样式和多媒体资源提供 `https:` 链接。
- **缩进**：统一使用 2 个空格缩进，不使用制表符，也不要混用空格和 Tab。
- **大小写**：所有 HTML 元素名、属性、属性值（非文本内容）、CSS 选择器与属性一律小写。
- **尾随空格**：移除所有多余尾随空格，避免无意义 diff。
- **编码**：使用 UTF-8（无 BOM），并在 HTML 文档 `<head>` 中声明 `<meta charset="utf-8">`。
- **注释与 TODO**：
  - 使用清晰的注释解释“为什么要这样做”。
  - 使用统一的 `TODO: 描述` 形式标记待办事项。

### HTML 结构与语义

- **文档类型**：统一使用 `<!doctype html>` 以确保处于标准模式（no-quirks mode）。
- **有效性**：尽量保持 HTML 结构有效，可使用 W3C HTML 校验工具检查。
- **语义化**：
  - 按用途使用元素：标题用 `h1–h6`，段落用 `p`，列表用 `ul/ol/li`，链接用 `a`，等。
  - 不用 `div` + JS 行为代替可语义表达的元素（例如按钮和链接）。
- **多媒体回退**：
  - 图片添加合适的 `alt` 文本。
  - 视频/音频提供字幕或文字描述（在条件允许时）。
- **关注点分离**：
  - HTML 只负责结构和语义，不混入样式（避免内联 `style`）和脚本逻辑。
  - 样式集中在 CSS 文件中，交互逻辑集中在 JS 中。
- **实体引用**：
  - 在 UTF-8 环境下，除 `<`、`&` 等特殊字符和控制字符外，避免使用实体引用。

### CSS 编写规则

- **有效性与校验**：保持 CSS 有效，利用校验工具查找无效或无效用的声明。
- **类命名**：
  - 使用含义明确或通用的类名（如 `.gallery`、`.login`、`.aux`），避免纯表现（如 `.button-green`）和无意义缩写（如 `.yee-1901`）。
  - 类名中单词之间使用连字符 `-` 分隔（如 `.video-id`），避免驼峰或下划线。
  - 在大型项目或可嵌入代码中为类名前添加产品/项目前缀（如 `.adw-header`）。
- **选择器使用**：
  - 避免 `id` 选择器，优先使用类选择器。
  - 除非必要，不用标签名 + 类组合（如 `ul.example`），以提升性能和可复用性。
- **声明习惯**：
  - 尽量使用简写属性（如 `padding: 0 1em 2em;`、`font: 100%/1.6 ...`）。
  - `0` 后省略单位（除非属性要求，如 `flex-basis: 0px;`）。
  - 在 -1 到 1 之间的小数前加 `0`（如 `0.8em`）。
  - 十六进制颜色可写为 3 位简写时采用简写（如 `#ebc`）。
  - 避免使用 `!important`，通过合理的选择器特异性解决覆盖问题。
  - 避免 UA 检测和 CSS hack，仅在确无替代方案时最后使用。

### CSS 格式化

- **声明顺序**：在缺乏自动化工具时，可按字母顺序排序属性，提升可维护性。
- **缩进与层级**：规则块内的内容缩进，嵌套结构（如 `@media` 内部）进一步缩进，直观反映层级。
- **分号与冒号**：
  - 每条声明以分号结尾。
  - 属性名与冒号之间无空格，冒号与属性值之间一个空格（如 `color: #333;`）。
- **选择器与规则块**：
  - 选择器与 `{` 之间留一个空格。
  - 多个选择器各占一行。
  - 规则块之间空一行。
- **引号风格**：
  - CSS 中，属性值和属性选择器使用单引号（如 `font-family: 'open sans', arial, sans-serif;`）。
  - `url()` 中不使用引号（如 `url(https://www.google.com/css/maia.css)`）。

## 一致性与本地风格

- **全局一致性**：上述规则构成了“全局词汇表”，让所有文档和组件在视觉和代码风格上保持统一。
- **本地一致性**：在修改既有文档时，应首先观察周围代码风格，并与之保持一致，即使它与“理想风格”略有差异。
- **演进方式**：在不破坏既有内容的前提下，逐步将旧内容迁移到统一风格；新内容需严格遵循上述规范。

> 总结：本系统将“严谨的编码风格指南”和“清晰的视觉层级与颜色系统”结合，通过统一的组件（标题、目录、代码块、按钮与表格）和明确的交互（展开/收起、锚点导航），为多语言、多技术栈的 Style Guide 提供了统一的设计与实现基础。
